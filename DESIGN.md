# 디자인 시스템 — 연구 기록 (Research Record)

포트폴리오의 주인공은 KCI 논문, 학회 우수논문상, 교육부장관상 등 **검증 가능한 실적**을 가진
AI 연구자다. 페이지의 단 하나의 임무는 "이 사람의 주장은 전부 기록으로 증명된다"를 보여주는 것.
그래서 페이지 전체를 **주석 달린 학술 문서**로 조판하고, 본인의 KCI 연구(YOLOv8 객체 검출)에서
가져온 **검출 프레임(detection frame)** 을 시그니처 모티프로 쓴다.

## 1. 컬러

액센트는 꿀 노랑 하나뿐이다. 대표 연구 두 건이 모두 이 팔레트의 세계에 있다:
꿀벌 응애 검출(벌 = 검정+노랑), 로드킬 분석(도로 표지 = 검정+노랑).

| 토큰 | 라이트 | 다크 | 용도 |
|------|--------|------|------|
| `--paper` | `#fbfaf7` | `#14130f` | 페이지 배경 (연속된 한 장의 종이) |
| `--surface` | `#ffffff` | `#1d1b15` | 호버 시 살짝 떠오르는 면 |
| `--ink` | `#1c1a15` | `#ece7da` | 본문 먹색 |
| `--ink-2 / --ink-3` | ink 68% / 45% | 동일 비율 | 보조/삼차 텍스트 |
| `--line / --line-soft` | ink 22% / 10% | ink 24% / 10% | 괘선 (헤어라인) |
| `--honey` | `#f0b100` | `#f2b705` | 검출 브래킷, 게이지, 마크 — 유일한 유채색 |
| `--honey-ink` | `#171407` (고정) | 동일 | 노랑 위 텍스트 |
| `--honey-text` | `#7a5f00` | `#edbc2e` | 본문 위 액센트 텍스트 (연도, 인용 링크) |

- 색 블록 교차 없음 — 문서는 한 장의 종이. 섹션 구분은 `--line` 괘선으로.
- `::selection`도 honey. 포커스 링도 honey.

## 2. 타이포그래피

| 역할 | 서체 | 용도 |
|------|------|------|
| 디스플레이 | **Noto Serif KR** 900 / 700 | 국문 논문 조판의 명조. 헤드라인, 장 제목, 실적 제목, 지표 숫자 |
| 본문 | **Pretendard** 400–600 | UI와 본문. 16px, line-height 1.7 |
| 데이터 | **IBM Plex Mono** 400–600 | 검출 라벨, 장 번호, 연도, 태그, 인용 링크, 사실 표 라벨 |

- 히어로: `clamp(2.3rem, 5.8vw, 3.7rem)` / 900 / lh 1.26 / `word-break: keep-all`
- 초록(소개 요약)은 명조 400 17px lh 1.95 — 논문 초록의 질감
- 모노 아이브로우: 12px, letter-spacing 0.16em, uppercase — `01 · ABOUT` 형식의 러닝 헤드

## 3. 시그니처: 검출 프레임

YOLO 검출 결과의 시각 언어를 그대로 인용한다.

- **히어로 사진**: honey 코너 브래킷 4개(3px, 22px) + 좌상단에 `연구자 · 0.99` 신뢰도 라벨
  (honey 배경, mono 11px). 페이지 로드 시 브래킷이 바깥에서 잠기는(`scale 1.14→1`) 애니메이션.
- **실적 목록**: 항목 호버/포커스 시 코너 브래킷(2.5px, 17px)이 나타나며 잠기고
  배경이 `--surface`로 떠오른다.
- 이 모티프 외의 장식은 모두 절제 — 굵기는 한 곳에만 쓴다.

## 4. 구조 장치

- **장(章) 번호**: 문서이므로 섹션은 `01 소개 → 05 연락처`로 번호를 매긴다 (순서가 정보다).
- **서지 형식 실적**: 프로젝트는 카드 그리드가 아니라 참고문헌 목록이다.
  번호 rail(46px) + 명조 제목 + mono venue 행(수상·연도·기관) + 기여 목록(honey 틱) +
  `#태그` + `[논문 ↗]` 꼴의 대괄호 인용 링크.
- **표**: 기술 숙련도(이름·honey 게이지·mono 등급), 신상 사실, 연락처 모두 괘선 행.
- **인용 링크** `.ref-link`: mono, 실제 대괄호 `[ ]`를 ::before/::after로 렌더.
  호버 시 honey 배경으로 반전. 미발행 자료는 `.ref-dead`(회색).

## 5. 버튼

- `.btn-ink`: 먹색 블록, 5px 라운드. **호버 시 honey로 반전** — 형광펜으로 칠한 듯한 전환.
- 그 외 CTA는 전부 `.ref-link` 인용 스타일.

## 6. 모션

- 페이지 로드: 히어로 요소가 `rec-rise`(16px 상승 페이드)로 시퀀스 등장(0 → 0.38s 딜레이),
  마지막에 검출 프레임이 잠기고 라벨이 뜬다.
- 스크롤 트리거 애니메이션 없음 — 문서는 조용해야 한다.
- `prefers-reduced-motion: reduce` 전면 존중.

## 7. 레이아웃

- 콘텐츠 최대폭 920px 단일 컬럼, 좌우 24px.
- 섹션 패딩 `72px / 84px`, 섹션 사이는 `--line` 괘선.
- 헤더: sticky, 종이색 88% + blur(14px), honey 사각 인장(`조`) + 명조 워드마크.
- 인쇄: 라이트 토큰 강제, 내비 숨김, 실적 항목 `break-inside: avoid` — 그대로 이력서가 된다.

## 8. Do / Don't

**Do**
- 유채색은 honey 하나만. 액센트가 하나라서 검출 라벨이 눈에 띈다.
- 정보 구분은 괘선과 여백으로. 그림자·테두리 카드 금지.
- 국문은 `word-break: keep-all`.
- 새 요소가 필요하면 문서의 어휘(서지, 표, 인장, 주석)에서 가져올 것.

**Don't**
- 두 번째 액센트 컬러 추가 금지.
- 색 블록으로 섹션 교차 금지 — 종이는 한 장이다.
- 검출 프레임을 아무 데나 남발 금지 — 히어로와 실적 호버, 두 곳뿐.
- 그라데이션, 글래스모피즘, 큰 라운드(>6px) 금지.
